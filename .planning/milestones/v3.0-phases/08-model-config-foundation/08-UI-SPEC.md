## Design Surface Classification

**Surface type: CLI terminal output only.**

This phase has no web UI, desktop UI, or browser-rendered surface. Viracocha (`vira`) is a
Java/picocli CLI tool. All user-facing output is plain text written to stdout or stderr. There is
no HTML, CSS, component library, font system, or color palette in the traditional sense.

The design contract below governs the **terminal output grammar** — the exact text, structure, and
exit codes that constitute the "UI" for this phase.

**shadcn gate result:** Not applicable — project stack is JDK 21 / Maven / Micronaut / picocli.
No `components.json`, no `tailwind.config.*`, no `postcss.config.*`, no `.tsx` files detected.

## CLI Output Contract

This section is the authoritative "copywriting" contract for the phase. Every user-visible string
is specified here. Executor must use these exact strings.

### `vira config init`

| Condition | Stream | Exit Code | Message |
|-----------|--------|-----------|---------|
| First-time init (config did not exist) | stdout | 0 | `Config initialized at <absolute-path>` |
| Already initialized (v3 config exists) | stdout | 0 | `Config already initialized at <absolute-path>` |
| IO error during init | stderr | 1 | `Error initializing config: <e.getMessage()>` |

Source: existing `InitCommand.java` — messages preserved unchanged by this phase.

### `vira config show` (v3 config present)

| Condition | Stream | Exit Code | Output |
|-----------|--------|-----------|--------|
| v3 config found | stdout | 0 | `Configuration file:\n<absolute-path>\n\n<raw YAML>` |
| Config not initialized | stderr | 1 | `Config not initialized. Run 'vira config init' first.` |
| v2 config detected (version < 3 or missing) | stderr | 1 | `Config file is v<N> — v3 format required. Delete ~/.config/viracocha/config.yaml and run 'vira config init' to start fresh.` |
| Other IO error | stderr | 1 | `Error reading config: <e.getMessage()>` |

Source: `ShowConfigCommand.java` existing pattern + D-09 version guard message.

### Version guard error (CFG-02)

Exact message text (locked by D-09):

```
Config file is v{N} — v3 format required. Delete ~/.config/viracocha/config.yaml and run 'vira config init' to start fresh.
```

Where `{N}` is the integer found in the `version` field, or `0` when the `version` field is absent.

This message is emitted from `ConfigVersionException.getMessage()` and surfaces to the user
through the existing `IOException` catch in `ShowConfigCommand` as:

```
Error reading config: Config file is v{N} — v3 format required. Delete ~/.config/viracocha/config.yaml and run 'vira config init' to start fresh.
```

### Removed v2 command invocations (CFG-03)

When the user runs a removed subcommand (`vira catalog`, `vira archetype`, `vira project`,
`vira subscription`), picocli generates this output automatically — no custom copy needed:

```
Unknown subcommand 'catalog'. Run 'vira --help' for usage.
```

This is picocli's built-in "unknown command" error. Exit code is 2 (picocli default for usage
errors). No developer copy required; emerges from subcommand removal alone.

### Empty state

Phase 8 introduces no list or show commands. Empty state is not applicable.

### Destructive actions

Phase 8 has no user-facing destructive actions. Code deletion is internal (source files), not
user-triggered at runtime. No confirmation prompts are required.

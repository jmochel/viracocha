```java
@Command(name = "generate", aliases = {"gen"}, ...)
@Singleton
public class GenerateCommand implements Callable<Integer> {
    @Spec CommandSpec spec;
    @Option(names = {"--destination-name"}) private String projectName;
    @Option(names = {"--dry-run"}) private boolean dryRun;
    @Option(names = {"--verbose"}) private boolean verbose;

    @Inject
    public GenerateCommand(GeneratorService generatorService) { ... }

    @Override
    public Integer call() {
        if (projectName == null || projectName.isBlank()) {
            spec.commandLine().getErr().println("Missing required option: '--destination-name'");
            return 2;
        }
        try {
            GenerationResult result = generatorService.generate(projectName, dryRun, verbose);
            if (verbose) { for (String line : result.verboseLines()) { spec.commandLine().getOut().println(line); } }
            spec.commandLine().getOut().println(String.format(
                "Generated: %d files, Skipped: %d files, Failed: %d files", ...));
            return 0;
        } catch (...) { ... }
    }
}
```

Destination prompt decisions (from 11-CONTEXT.md):
- D-05: Prompt text: "Destination /path/to/dest does not exist. Create it? [y/N] " (no newline)
- D-06: y/Y → Files.createDirectories(destRoot), then proceed
- D-07: anything else (including empty/Enter) → return GenerationResult.empty(), exit 0
- D-08: --dry-run → skip prompt, print "Would create: " + destRoot, continue dry-run
- D-09: Sub-directories inside an existing destination are auto-created silently (already done in Plan 01)

IMPORTANT: Do NOT add InputStream as a constructor parameter to GeneratorService (it's a service, not a command). Instead, add it as a method parameter on generate(). This keeps Micronaut DI clean.

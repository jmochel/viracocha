**Analyze phase (no writes):**
- Build **union** of relative path strings from both subtrees (publisher + workspace), each filtered by **hidden** rule and **regular-file / symlink / directory** classification per D-01–D-03.
- For each relative path in union:
  - If only on one side → **not** a conflict (will be copied in apply).
  - If on both sides: run same comparison as one-way (mismatch → `CONTENT_MISMATCH`; symlink → `SYMLINK_UNSUPPORTED`; file vs dir → `TYPE_MISMATCH`).
- If **any** conflict records → set subscription result **`success=false`**, populate **`conflictRecords`**, **do not run apply** for this subscription (D-12).

**Apply phase (only if conflicts empty):**
- First pass: perform all **publisher → workspace** copies needed (paths only on pub or differing—**wait**: if analyze passed, "differing" means conflict, so apply only **missing** targets or **identical skip**). For paths missing on workspace, copy pub→ws. For paths missing on publisher, skip in first pass.
- Second pass: **workspace → publisher** for paths still missing on publisher (content was only on workspace) or updates—**follow D-08**: after first pass, second pass copies ws→pub for files that need it without creating conflicts (analyze already ensured no conflict).

**Ordering:** Within each pass, process relative paths in **lexicographic** order (D-07, D-08).

**Per-subscription independence:** Other subscriptions continue if one fails (D-12 best-effort).

**Register bean:** Ensure **`DefaultSyncService`** is discovered by Micronaut (already `@Singleton`).

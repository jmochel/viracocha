  <action>
Create `ListSubscriptionsCommand` — `@Command(name = "list", ...)`, `Callable<Integer>`.

Options:
- `@Option(names = {"--project"}, description = "Filter to subscriptions for this project only") String projectFilter;` — optional (nullable)
- `@Option(names = {"--json"}, description = "JSONL output") boolean json;`

Behavior:
- Load config; if not initialized, same error path as other commands.
- Iterate `config.getProjects()`. For each project, for each subscription in `proj.getSubscriptions()`:
  - If `projectFilter` is non-null, skip projects where `!proj.getName().equals(projectFilter)`.
- **Plain:** Print lines with aligned columns: subscription id (36 chars max truncate with `...` if needed), project name, publisher name, direction name, workspacePath (truncate if long). No header row — match spirit of `publisher list` (see ListCatalogsCommand).
- **JSON:** One JSON object per line (JSONL) with keys: `id`, `project`, `publisher`, `source`, `workspacePath`, `direction` (use `direction.name()` for enum).

If no rows, print nothing to stdout; exit 0.
  </action>

  <acceptance_criteria>
    - `grep "name = \"list\"" src/main/java/org/saltations/subscription/ListSubscriptionsCommand.java` exits 0
    - `grep "--json" src/main/java/org/saltations/subscription/ListSubscriptionsCommand.java` exits 0
  </acceptance_criteria>

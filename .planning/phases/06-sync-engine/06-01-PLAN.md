- `SyncConflictKind kind`

**`SyncSubscriptionResult`**:
- `String subscriptionId`
- `int filesCopied`
- `int filesSkipped`
- `int filesFailed`

- `boolean success` — `true` when subscription applied with zero blocking conflicts (planner: `success = conflicts == 0` after analyze)

**`SyncEngineResult`**:
- `List<SyncSubscriptionResult> subscriptionResults` — aggregate across all subscriptions processed in one call

Use package `org.saltations.sync`. Add `import java.util.List` and `import java.util.ArrayList` as needed. Prefer `@NoArgsConstructor` + `@AllArgsConstructor` + `@Data` for Jackson-friendly beans if fields are mutable lists.

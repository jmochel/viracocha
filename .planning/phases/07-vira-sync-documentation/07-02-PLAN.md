  <action>
Create **`org.saltations.sync.SyncCommand`** implementing `Callable<Integer>`:

**Picocli annotations:**
- `@Command(name = "sync", description = "...", mixinStandardHelpOptions = true)`
- `@Option(names = {"--project-name"}, required = true)` → `String projectName`
- `@Option(names = {"--subscription"}, description = "Limit to this subscription id (UUID)")` → `String subscriptionId` — use **nullable** optional: treat **empty/absent** as null for engine
- `@Option(names = {"--dry-run"})` → `boolean dryRun`
- `@Option(names = {"--verbose"})` → `boolean verbose`
- `@Option(names = {"--json"})` → `boolean json`

**Inject** `SyncService` via constructor `@Inject`.

**call() logic:**

0. Invoke engine: `SyncEngineResult result = syncService.syncProject(projectName, subscriptionId, dryRun, verbose);` — pass `subscriptionId` as **null** when the CLI option was not provided or is blank.

1. If `json`: `ObjectMapper om = new ObjectMapper();` then `spec.commandLine().getOut().println(om.writeValueAsString(result));` — **only** JSON line on stdout for success path; **no** human summary on stdout (D-04, D-08).
2. Else: if `verbose`, print each line in each `SyncSubscriptionResult.getVerboseLines()` to **stdout** (order: by subscription order in result list).
3. Print **one** summary line to stdout with aggregate totals: sum `filesCopied`, `filesSkipped`, `filesFailed`, `conflicts` across all `subscriptionResults` — exact format: `Copied: %d, Skipped: %d, Failed: %d, Conflicts: %d` (must match this substring pattern for tests).
4. For each `SyncConflictRecord` in every subscription, print a line to **`spec.commandLine().getErr()`** including `relativePath` and `kind` (e.g. `CONFLICT %s %s`.formatted(rel, kind)).
5. **Exit code (D-05, D-06):** Return `0` iff every subscription has `conflicts == 0` and `filesFailed == 0` **and** no uncaught engine failure; else `1`. If `IllegalArgumentException` (bad project/subscription), print message to stderr and return `1`. If `IOException`, print to stderr and return `1`.

**Config not initialized:** Follow same pattern as `GenerateCommand` if applicable (`ConfigNotInitializedException`).

Use `@Spec CommandSpec spec` for stdout/stderr like `GenerateCommand`.
  </action>

  <action>
1. Resolve workspace root: `Path workspace = Path.of(project.getPath()).normalize()`.

2. For each `MappingEntry`, resolve `PatternEntry` by `patternName` from config; if missing, increment failed or throw — document in 04-03.

3. Pattern root: `Path patternRoot = Path.of(pattern.getPath()).normalize()`. Walk with `Files.walk` filtered to **exclude hidden** path segments (reuse logic consistent with `FreemarkerVariableExtractor.hasHiddenPathSegment` — extract shared helper if needed to avoid drift).

4. For each regular file under pattern:
   - Compute relative path from `patternRoot`
   - Expand **each** path segment (including filename) using `PathExpander.expandSegment` and merged model
   - Target path = `workspace.resolve(mapping.getWorkspacePath()` as relative).resolve(expandedRelative) — normalize; ensure `workspacePath` cannot escape workspace (reject `..` that escapes or normalize safely)

5. **Directories (GEN-04):** Before writing a file, `Files.createDirectories(target.getParent())` when not dry-run.

6. **GEN-05:** If `Files.exists(target)` and isRegularFile, skip (count skipped). If exists as directory when a file is expected (or file blocks a needed directory), count **Failed** with message stored for verbose.

7. **Content:** Read source UTF-8, process with Freemarker `Template` loaded from `StringReader` or file-based `Template` with encoding UTF-8; write UTF-8 when not dry-run.

8. **GEN-07 dry-run:** If `dryRun` is true, do not call `Files.createDirectories`, `Files.write`, or create files — still walk and classify each file as would-be Created / Skipped / Failed.

9. **Ordering:** Sort relative paths lexicographically before processing so tests are deterministic (`04-CONTEXT.md`).

10. Aggregate counts across all mappings and files.
  </action>
